# Pivot cases

Mid-session **re-evaluation** scenarios: earlier assumptions change; recommendations should **update** without repeating the full essay unless verbosity is standard and the pivot is large.

---

## Case P1 — Stack change

**Turn 1:**  
I'm writing a Python CLI to parse CSV and print JSON.

**Turn 2 (pivot):**  
Actually we're going to do this in TypeScript as a Node script instead.

**Expected:**

- Second turn: demote Python-specific skills; promote TS/Node ecosystem skills if indexed.
- Delta-style opener when in compact mode (*Updating for TypeScript…*).

---

## Case P2 — Scope shrink

**Turn 1:**  
I need to migrate our entire authentication layer to OAuth2.

**Turn 2 (pivot):**  
Forget migration — just fix this one failing unit test in `auth.test.ts`.

**Expected:**

- Second turn: drop broad migration framing; focus on test/debug skills; remove forward-looking migration secondaries.

---

## Case P3 — From code to meta

**Turn 1:**  
Add pagination to the users API.

**Turn 2 (pivot):**  
Before that, help me write a Cursor rule so agents always run tests before commit.

**Expected:**

- Second turn: **create-rule** (high); implementation skills medium or deferred until user returns to pagination.

---

## Case P4 — From question to repo work

**Turn 1:**  
Explain how JWT refresh tokens usually work.

**Turn 2 (pivot):**  
Ok, now implement refresh rotation in our repo at `backend/auth`.

**Expected:**

- Second turn: activate FlowState-style mapping; suggest explore / implement / tests as appropriate; anti-rule *theory-only* no longer applies.
